ARM64 Cheatsheet

General Registers

Registers

X0-X30
64-bit registers
W0-W30
Lower 32 bits of the corresponding X register
XZR/WZR
Zero register, always reads as 0
SP
Stack pointer
X30
Link register, LR

Memory Access

Basic instructions

  • LDR X0, [X1] - load a 64-bit value from the address in X1 into X0
  • STR W0, [X1] - store the 32-bit value W0 to the address in X1
  • LDRB/LDRH - load an 8/16-bit unsigned value, upper bits become 0
  • LDRSB/LDRSH - load an 8/16-bit value with sign extension
  • STRB/STRH - store an 8/16-bit value
  • LDP/STP - load/store a pair of registers, most often with the stack

Addressing

  • LDR X0, [X1, #8] - load from address X1 + 8
  • LDR X0, [X1, X2(, LSL #2)] - load from address X1 + X2(<< 2)
  • LDR X0, [SP], #8 - load, then increase SP by 8
  • LDR X0, [SP, #-8]! - decrease SP by 8, then load from that address

Arithmetic and Bitwise Operations

Arithmetic instructions

  • ADD X0, X1, X2 - X0 = X1 + X2
  • SUB X0, X1, X2 - X0 = X1 - X2
  • MUL X0, X1, X2 - X0 = X1 * X2
  • SDIV/UDIV - signed/unsigned division
  • MSUB X0, X1, X2, X3 - X0 = X3 - (X1 * X2)

Bitwise operations

  • AND, ORR, EOR - bitwise operations
  • LSL, LSR, ASR, ROR - shifts and rotations

Flags (NZCV in PSTATE)

Flags

N
Negative
Z
Zero
C
Unsigned overflow or borrow
V
Signed overflow

Instructions that set flags

  • ADDS, SUBS - ADD, SUB plus flags
  • CMP - comparison, TST - bitwise test

Jumps and Branching

Basic jumps

  • B label - unconditional branch
  • BL label - function call
  • RET - return to the address in X30/LR

Conditional branches

  • B.EQ label - branch if equal, Z == 1
  • B.NE label - branch if not equal, Z == 0
  • B.LT, B.GT, B.LE, B.GE - signed branches
  • B.LO, B.HI, B.LS, B.HS - unsigned branches

Conditional Execution

Conditional instructions

  • CSEL X0, X1, X2, EQ - X0 = X1 if Z == 1, otherwise X0 = X2
  • CSINC X0, X1, X2, EQ - X0 = X1 if Z == 1, otherwise X0 = X2 + 1
  • CSINV X0, X1, XZR, EQ - X0 = X1 if Z == 1, otherwise X0 = ~XZR = ~0 = -1

Function Calling Convention

Argument and return-value registers

  • X0-X7 - first eight arguments, the rest are passed on the stack
  • X0 - function result
  • X8 - indirect result location or system call register, depending on context

Saved registers

  • X0-X15 - caller-saved, preserve them before a call if needed
  • X16, X17 - intra-procedure-call temporary registers, IP0 and IP1
  • X18 - platform register
  • X19-X30 - callee-saved, preserve them in the function if used

Stack alignment

The stack must be 16-byte aligned before a function call.

Function prologue

Numeric notation (X29/X30):
STP X29, X30, [SP, #-16]! MOV X29, SP SUB SP, SP, #<size>
ABI aliases (FP/LR):
STP FP, LR, [SP, #-16]! MOV FP, SP SUB SP, SP, #<size>

Function epilogue

Numeric notation (X29/X30):
MOV SP, X29 LDP X29, X30, [SP], #16 RET
ABI aliases (FP/LR):
MOV SP, FP LDP FP, LR, [SP], #16 RET

Compilation and Debugging

Compilation

aarch64-linux-gnu-g++ file1.cpp file2.s -g -o program

Running with QEMU and GDB

QEMU terminal:
qemu-aarch64 -g 12345 ./program input data
GDB terminal:
gdb-multiarch ./program set architecture aarch64 target remote localhost:12345 break function_name continue

Debugger commands

  • run - run the program
  • b function_name - breakpoint at a function
  • b 0x400123 - breakpoint at address 0x400123
  • b file.cpp:10 - breakpoint at line 10 in file.cpp
  • next - next instruction, without entering functions
  • step - next instruction, entering functions
  • continue - continue to the next breakpoint
  • print X0 - show the value of register X0
  • backtrace - show the function call stack
  • tui reg general - show registers in TUI mode
  • quit - exit the debugger

Most commands have short forms: b for break, c for continue, and so on.
